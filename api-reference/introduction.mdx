## Authentication

All API endpoints are authenticated using Bearer tokens and picked up from the specification file.

```json
"security": [
  {
    "bearerAuth": []
  }
]
```

## Model Context Protocol (MCP)

MCP is enabled globally for all endpoints through the `x-mint` extension at the root level of the OpenAPI specification. You can selectively disable MCP for specific endpoints by adding the `x-mint` extension at the operation level.

### Global Configuration

Enable MCP for all endpoints by default:

```json
{
  "openapi": "3.1.0",
  "x-mint": {
    "mcp": {
      "enabled": true
    }
  },
  // ...
}
```

### Per-Endpoint Configuration

Disable MCP for a specific endpoint by overriding the global setting:

```json
{
  "paths": {
    "/plants/{id}": {
      "delete": {
        "x-mint": {
          "mcp": {
            "enabled": false
          }
        },
        // ...
      }
    }
  }
}
```

In this example, MCP is enabled globally, but disabled for the `DELETE /plants/{id}` endpoint.

## Using MCP with AI Assistants

When MCP is enabled, your documentation automatically exposes an MCP server at `https://yourdocs.com/mcp`. This allows AI assistants like Claude, Cursor, and Goose to:

- **Understand your API**: Read endpoint descriptions, parameters, and schemas
- **Generate code**: Create working examples in multiple programming languages
- **Answer questions**: Provide accurate information about your API
- **Help users**: Guide developers through API interactions

### Connecting to the MCP Server

To use MCP with your AI assistant:

1. **Find your MCP server URL**: Your documentation's MCP server is available at `https://yourdocs.com/mcp`
2. **Configure your AI client**: Add the MCP server URL to your AI assistant's configuration
3. **Start asking questions**: Ask natural language questions about your API endpoints

### Example Questions

With MCP enabled, you can ask your AI assistant:
- "How do I fetch all plants?"
- "What parameters does the GET /plants endpoint accept?"
- "Show me how to create a new plant using curl"
- "What's the schema for a Plant object?"

The AI assistant will use the MCP tools to access your OpenAPI specification and provide accurate, up-to-date answers.

## Using MCP in Your Development Project

As an engineer working in another project, you can connect Cursor to the MCP server to access your API documentation directly from your codebase. This enables AI-assisted development with real-time access to endpoint specifications.

### Step 1: Configure Cursor with MCP Server

In your development project (not the docs repository), you can set up the MCP connection using the Mintlify CLI or manually.

#### Option A: Using Mintlify CLI (Recommended)

The easiest way is to use the Mintlify MCP CLI tool:

```bash
npx mint-mcp add <your-subdomain-or-domain>
```

Replace `<your-subdomain-or-domain>` with your Mintlify documentation's subdomain or domain (e.g., `docs.yourcompany.com` or `yourcompany.mintlify.app`).

This command automatically:
- Generates the MCP server configuration
- Adds it to your Cursor settings
- Sets up the connection to your documentation's MCP endpoint

#### Option B: Manual Configuration

Alternatively, you can manually configure the MCP server:

<Tabs>
<Tab title="macOS/Linux">

Create or update `~/.cursor/mcp.json`:

```json
{
  "mcpServers": {
    "your-docs": {
      "url": "https://yourdocs.com/mcp"
    }
  }
}
```

</Tab>

<Tab title="Windows">

Create or update `%APPDATA%\Cursor\mcp.json`:

```json
{
  "mcpServers": {
    "your-docs": {
      "url": "https://yourdocs.com/mcp"
    }
  }
}
```

</Tab>
</Tabs>

<Tip>
Replace `https://yourdocs.com/mcp` with your actual documentation URL. The MCP server is automatically available at `/mcp` on your published documentation site.
</Tip>

### Example Workflow

Here's a complete example of using MCP in a development workflow:

**1. Starting a new feature that uses the API:**

```typescript
// In your project file
// Ask Cursor: "Show me how to fetch plants with pagination"

// Cursor generates code using MCP to understand the API:
async function fetchPlants(limit?: number) {
  const response = await fetch('https://api.example.com/plants?limit=' + (limit || 10), {
    headers: {
      'Authorization': `Bearer ${process.env.API_KEY}`
    }
  });
  
  if (!response.ok) {
    throw new Error(`API error: ${response.status}`);
  }
  
  return response.json();
}
```

**2. Creating a new resource:**

```typescript
// Ask Cursor: "Create a function to add a new plant with the correct schema"

// Cursor uses MCP to get the exact schema:
interface NewPlant {
  name: string;  // Required
  tag?: string;  // Optional
  id: number;    // Required
}

async function createPlant(plant: NewPlant) {
  // Implementation generated from MCP schema
}
```

### Benefits of Using MCP in Development

- **Always up-to-date**: Cursor reads directly from your published OpenAPI spec
- **Type safety**: Generate TypeScript types that match your actual API
- **Error prevention**: Cursor knows which fields are required and their types
- **No context switching**: Access API docs without leaving your editor
- **Accurate examples**: Code examples are generated from your actual API specification

### Troubleshooting

<Warning>
**MCP server not found**: Ensure your documentation is published and accessible. The MCP server is only available on published documentation sites, not local previews.
</Warning>

If you're having connection issues:

1. Verify the MCP server URL is correct: `https://yourdocs.com/mcp`
2. Check that your documentation site is live and accessible
3. Ensure MCP is enabled in your OpenAPI specification
4. Restart Cursor after configuration changes
